# 电商系统架构说明

电商系统由网关层、业务服务层、数据层和消息层组成。

网关层包含 API Gateway，负责统一鉴权、限流和路由。

业务服务层包含用户服务、订单服务、库存服务和支付服务。
用户服务提供登录、注册、用户信息查询功能，读写用户数据库。
订单服务提供创建订单、取消订单、订单查询功能，会调用用户服务校验用户信息，并调用库存服务扣减库存。
订单服务读写订单数据库。
库存服务读写库存数据库。
支付服务调用第三方支付平台完成付款。

消息层使用 Kafka。
订单服务通过 Kafka 发布订单创建事件。
库存服务订阅订单创建事件。

数据层包含用户数据库、订单数据库、库存数据库和 Redis 缓存。
订单服务依赖 Redis 缓存订单查询结果。
